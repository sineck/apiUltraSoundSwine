# Documentation Map

This folder keeps small reference notes only. The runtime source of truth remains the live code, `config/.env`, Docker files, and focused tests.

## Files

- `api.txt`: quick local URL reference for API docs and the Gemini detection form.
- `function.txt`: quick function map for the Gemini image-detection flow.

## Boundaries

- Deployment, Docker commands, troubleshooting, and operational notes belong in `README-Docker.md`.
- Runtime configuration belongs in `config/.env`; `config/.env.example` is only a template/mockup.
- Agent working rules belong in `AGENTS.MD`.
- Experimental anomaly-detection notes belong under `AnomalyDetection/` and are not the active API deployment guide.
